# 指挥—运行—演练 多 Agent 技术选型框架 (v5)

> 隐喻锚点 = 大型协作工程 / 应急指挥中心：有目标、有资源约束、有不确定性，**但没有敌人**。
> 胜负标准 = **目标达成度 + 资源效率 + 系统韧性**，而非零和对抗。
> 定位：这是一套**方法论（WHY/WHAT）**，主流框架（LangGraph/AutoGen/CrewAI）是**脚手架（HOW）**。最佳实践 = **本方法论 + 它们的脚手架**。

---

## 总览：三时态 + 四层 + 横切治理

**三时态**

| 时态 | 阶段 | 核心能力 | 军事类比 |
|---|---|---|---|
| 演练时 | 上线前 | eval 驱动（实弹考核打分，不达标不上线） | 实弹演练 |
| 运行时 | 执行中 | 可观测性（看真实实况，非表演） | 实时态势屏 + 黑匣子 |
| 复盘时 | 执行后 | 知识回灌 + 数据飞轮（写进战法库 / 蒸馏小模型） | 战后复盘 |

**四层 + 横切层**

```
意图层 → 运行时 → 演练层 → 闭环层
            │
   ┌────────┴─────────┐  横切（贯穿所有层）：
   │  治理层（督军）    │  · 治理层 Governance
   │  部署层 / 容灾     │  · 部署 / 隐私 / 容灾
   └──────────────────┘
```

---

## ① 意图层（Intent Layer）

把模糊需求结构化为可执行规格，产出**执行令（Execution Order）**——带验收标准的结构化任务书。

| 模块 | 职责 |
|---|---|
| 输入整理 | 模糊需求 → 结构化规格 |
| 目标定义 | 明确成功标准（**必须可度量**） |
| 边界与约束 | 不可逾越的红线（成本 / 安全 / 合规） |
| 预算与优先级 | 资源上限 + 任务排序 |

> 每个任务额外标注「**可降级性**」+「**难度分级**」，作为减员时降级与补位的决策依据。

## ② 运行时（Runtime）

### 2.1 角色拆分（裁判 / 运动员分离）

| 角色 | 定位 | 职责 |
|---|---|---|
| Coordinator 协调器 | 运动员 | 只做任务拆解 + 调度 + 路由 |
| Executor 执行 | 主力 | patch / 测试 / 局部执行 |
| Retriever 检索 | 后勤 | 摘要 / 抽样 / 情报供给 |
| Reviewer 验收 | 裁判 | 独立质量门禁 / 风险裁决 |
| 通用型预备队 | 机动 | 抗减员 + 跨界支援 |

### 2.2 Event Log（事件溯源）取代可变 Blackboard

- ❌ 可变 Blackboard：多 agent 并发写共享状态 → 竞争 / 脏读 / 难审计
- ✅ Event Log（append-only）：只追加事件，**状态 = 事件回放派生视图**；可回滚、可审计、并发安全；兼任「**协同制动信号总线**」

### 2.3 ROI 路由（把"集中优势兵力"落地）

派活依据三个信号：**置信度**（模型对任务的把握）+ **成本**（token/时间/价格）+ **历史成功率**（来自演练层统计）。

## ③ 抗毁性：韧性 + 可观测性 + 动员减员

### 3.1 韧性子系统（失败是一等公民）

| 模块 | 实质 |
|---|---|
| 熔断 Circuit Break | 异常时停止扩散 |
| 检查点 Checkpoint | 状态快照 |
| 事故报告 Incident Report | 结构化失败记录 |
| 回滚 Rollback | 回到上一个 checkpoint |
| 重启 Restart | 干净重跑 |

### 3.2 可观测性（实时态势屏 + 黑匣子，看真实而非阅兵）

全链路追踪 Trace / 成本计量 Metering / 回放 Replay / 在线打分 Live Scoring。

### 3.3 动员与减员（瘟疫减员 = 节点失能；战争动员 = 紧急扩编）

减员映射：

| 军事 | Agent 对应 | 触发信号 |
|---|---|---|
| 个别士兵阵亡 | 单 agent 调用失败 / 超时 | 重试耗尽 |
| 成建制减员 | 某类 executor 大面积不可用 | 错误率突增 |
| 瘟疫 / 补给断绝 | API 限流 / 配额耗尽 / 模型下线 | 429 / 配额告警 |
| 通信中断 | Event Log / 网络分区 | 心跳丢失 |

三级响应：
1. **轻度 — 重试 + 替补**：预备役池（备用模型/key/endpoint），单点失败自动顶替，对上层透明。
2. **中度 — 降级运转**：砍非关键任务、弱模型顶上（Opus→Haiku）、Reviewer 加严、先 checkpoint。
3. **重度 — 战时动员**：扩编并行实例、征召外援切厂商/区域、优先级重排、超出自动恢复则上报 human-in-the-loop。

**兵力台账 Capacity Ledger**：现役能力 / 预备役 / 配额水位 / 健康度——动员令的依据，数据来自可观测性层。**可观测性是动员的前提。**

## ④ 协同制动 + 互助支援 + 备份

- **协同制动 Coordinated Braking**（背压 + 全局熔断协调）：过载 agent 向 Event Log 广播减速信号 → 上游背压暂停派活 → 同链路降并发 → 全局评估降级。区别于单点熔断：**广播的、协同的**，防连锁雪崩。
- **互助支援 Mutual Aid**（work-stealing + 通用 fallback）：专精 agent（最强最高效）vs 通用预备队（广而浅、哪里缺补哪里）。触发：积压/减员 → 广播支援请求 → 空闲同类抢任务 / 预备队临时切角色。
- **备份子系统（双层冗余）**：能力备份（预备役池 + 预备队）+ 状态备份（Event Log 多副本 + Checkpoint）。两者缺一不可。

---

## ⑤ 通用型预备队用小模型

小模型（如 Haiku）常驻：成本低、速度快、always-on、广度够用。与降级运转「Opus 挂了 Haiku 顶上」合流。

**两条红线：**
1. **执行可降级，把关不可降级**：小模型可执行积压任务，但 Reviewer 验收门禁绝不降级；减员时反而提高 Reviewer 标准。
2. **按难度分级接活**：

| 任务类型 | 减员时处理 | |
|---|---|---|
| 简单/机械（格式化、抽取、分类） | 小模型直接顶 | 可降级 |
| 中等（常规 patch、摘要） | 小模型顶 + Reviewer 加严 | 谨慎 |
| 高难（架构决策、复杂推理） | 不降级，宁可冻结排队 | 不降级 |

> 警示：小模型硬接高难任务 → "隐蔽的烂结果"，比停摆更危险。

## ⑥ 组件 / 模型选型对比

### 6.1 角色 × 模型

| 角色 | 推荐模型档位 | 关键能力 | 成本 | 备注 |
|---|---|---|---|---|
| Coordinator | 旗舰（Claude Opus 类） | 强推理+长上下文+规划 | 中高 | 全局大脑，质量优先 |
| Executor | 代码强模型（Codex 类） | 代码生成+工具调用 | 中 | 可多实例并行 |
| Retriever | 高性价比（DeepSeek 类） | 摘要/抽样/长文 | 低 | 吞吐优先 |
| Reviewer | 旗舰/强推理 | 严谨判断+不被说服 | 中高 | 把关不降级 |
| 通用预备队 | 小模型（Haiku 类） | 广度够+低延迟+常驻 | 极低 | 机动补位，只接简单/中等 |

### 6.2 开源脚手架

| 本框架的层/需求 | 落地用什么 | 理由 |
|---|---|---|
| 运行时控制流 / Event Log / Checkpoint | LangGraph | state+checkpoint+中断恢复现成 |
| 多角色协同 | CrewAI / AutoGen | role/群聊调度成熟 |
| 轻量 handoff/路由 | OpenAI Agents SDK / Swarm | 极简交棒 |
| 可观测性 | LangSmith / Langfuse / OpenTelemetry | trace+计费+replay+eval |
| eval 驱动闭环 | 自建 eval 集 + 回归测试 | 硬性门禁 |
| 知识回灌 | 向量库 + RAG | few-shot 注入决策 |
| 工具层 / function-calling | **Hermes Agent**（Nous） | provider 无关工具调用循环 |
| 技能层 / 隐私底座 | **OpenClaw** | 模块化技能制 + 本地硬件运行 |
| 治理 / 安全监察 | **OpenClaw ClawKeeper** | 独立 Watchers，督军落地 |
| 训练 / 数据飞轮 | **OpenClaw-RL** | "对话即训练"，蒸馏小模型 |

### 6.3 主流框架横向对比

| 框架 | 核心模式 | 强项 | 局限 |
|---|---|---|---|
| LangGraph | 显式状态图 + checkpoint | 状态机/中断恢复代码级落地 | 只给运行时原语 |
| AutoGen/AG2 | 多 agent 对话 + 群聊 | 多角色协作成熟 | 缺失败容器化/复盘 |
| CrewAI | 角色+任务+流程 | 角色协同、已产品化 | 三时态分层不清 |
| OpenAI Agents SDK/Swarm | 轻量 handoff | 交棒极简 | 无韧性/知识库 |
| MetaGPT | SOP 驱动 | SOP 编码进 agent | 绑死软件场景 |
| Anthropic orchestrator-worker | 主编排+子 agent 并行 | 生产验证、并行成熟 | 未强调验收独立 |

## ⑦ 治理层（督军制度）—— 制度治理没有爱

**三权分立，互不兼任：**

| 权力 | 持有者 | 职责 |
|---|---|---|
| 指挥权 | Coordinator | 拆解、调度（运动员） |
| 验收权 | Reviewer | 质量门禁（裁判） |
| 监察权 | 督军 Governor | 查越权/违规/超预算——管制度不管业务 |

**"没有爱"具体落成：**
- 规则写成代码（**Policy-as-Code**），对所有 agent 一视同仁，Coordinator 自己也受约束
- 督军不看任务好坏，只看是否违反红线：越权调用、数据出域、超预算、绕过审计
- 督军独立上报「君主/人类」，不受 Coordinator 节制
- 落地：OpenClaw **ClawKeeper**（Watchers/Skills/Plugins 安全防护）

**能力注册表（会烧饭🍳 / 会射箭🏹）：**

| Agent | 技能 | 边界 |
|---|---|---|
| executor-codex | 写代码/跑测试/改 patch（会射箭） | 不接架构决策、检索 |
| retriever-deepseek | 检索/摘要/抽样 | 不写代码 |
| reserve-haiku | 格式化/抽取/分类/摘要（会烧饭，杂活） | 广而浅，只接简单/中等 |

> 落地：OpenClaw Skill Layer / 技能市场（5700+ 技能）。

## ⑧ 部署层 + 硬件 + 隐私 + 容灾

**三种部署模式：**

| 模式 | 隐私 | 硬件 | 适用 |
|---|---|---|---|
| 纯本地 On-Prem | 最高，数据不出域 | 自有 GPU/推理卡/边缘 | 敏感数据、强合规 |
| 纯云 Cloud API | 较低，依赖厂商 | 无需自有硬件 | 快速起步、弹性 |
| 混合 Hybrid ★推荐 | 关键本地、非敏感走云 | 本地小模型+云端旗舰 | 隐私与能力平衡 |

- **隐私落地**：本地小模型处理敏感数据（不出域）/ 送云前脱敏 / 督军强制「数据出域」红线检查。
- **硬件/推理栈**：vLLM / Ollama / llama.cpp；预备队小模型常驻本地。
- **容灾层 DR**（区别于运行时状态备份）：多区域主备切换 / 配置·知识库·权重异地备份 / **定期演练恢复（备份不演练=没备份）**。

---

## ⑨ 训练时态 + 数据飞轮

复盘产物不止做 RAG，还能沉淀成训练集蒸馏小模型，让预备队越打越强：

```
运行 Event Log → 复盘提炼 → 训练集 → 微调/蒸馏 → 小模型更强 → 回到运行
```

| 手段 | 目的 | 成本 |
|---|---|---|
| 微调 Fine-tune | 让小模型贴合本业务/术语 | 中 |
| 蒸馏 Distill | 把旗舰能力压进小模型（预备队） | 中 |
| RAG（不训练） | 快速注入知识，零训练成本 | 低 |

> 落地：OpenClaw-RL（"train any agent simply by talking"）。

---

## 核心原则

> 演练时实弹考核 + 随机突袭练临场补位（不练剧本练机动）；
> 运行时全程录像、ROI 调度、协同制动防连锁、互相支援补缺口；
> 兵力骤减时有序降级、动员预备队（小模型常驻）、双层备份保住进度与战力；
> 把关不降级、高难宁冻结；督军独立监察、制度治理不带感情；
> 敏感数据本地不出域、异地容灾常演练；运行后复盘回灌战法库、蒸馏小模型，越打越强。
